# 待解問題

本文件只放尚未驗證或尚未定案，但足以影響後續實作的問題。

## 高優先

### Q-001 範例網址是否可直接取得內容？

- 狀態：已驗證
- 影響：
  若無法直接抓取，後續需要調整抓取方式、標頭或節流策略。
- 建議行動：
  已驗證可直接取得內容，HTTP 200。

### Q-002 頁面編碼是什麼？

- 狀態：已驗證
- 影響：
  直接影響中文欄位與表格解析是否正確。
- 建議行動：
  已驗證 header 與 meta 都指向 `big5`。

### Q-003 主要資料是否直接存在 HTML 內？

- 狀態：已驗證
- 影響：
  若資料是前端二次請求或腳本動態載入，parser 與 fetcher 設計都要改。
- 建議行動：
  已驗證主要資料表存在於 HTML 內，可直接解析。

### Q-004 URL 參數 `a`、`b`、`c`、`d` 的意義是什麼？

- 狀態：部分驗證
- 影響：
  影響批次抓取與查詢輸入設計。
- 建議行動：
  目前已確認：
  - `a`：券商代碼
  - `b`：分行代碼
  - `c=E/B`：張數 / 金額模式
  - `d=1/5/10/...`：近 N 日
  仍待驗證：
  - `e`、`f`：自訂日期區間
  - 特殊分點 code 的完整行為

### Q-005 主要表格欄位有哪些？

- 狀態：已驗證
- 影響：
  直接影響 schema 與 cleaner 實作。
- 建議行動：
  已驗證主表為兩個巢狀 table：
  - `買超`
  - `賣超`
  欄位為：
  - 第一欄表頭顯示「券商名稱」，但資料列實際是股票
  - 第二欄與第三欄依模式顯示 `買進金額/賣出金額` 或 `買進張數/賣出張數`
  - 第四欄為 `差額`

## 中優先

### Q-006 資料日期從哪裡取得？

- 狀態：已驗證
- 影響：
  `trade_date` 是核心欄位，若頁面未顯示需另尋來源。
 - 結論：
   可從頁面文字 `資料日期：YYYYMMDD` 取得。

### Q-007 買入 / 賣出 / 淨額是頁面直接提供還是需要推導？

- 狀態：已驗證
- 影響：
  影響清洗公式與欄位可信度。
 - 結論：
   頁面直接提供買進、賣出與差額。賣超區塊的差額目前驗證為負值，但 parser 仍保留保底邏輯，避免未來頁面改成絕對值時失真。

### Q-008 後續批次抓取的輸入是 URL 清單還是券商代碼清單？

- 狀態：未定案
- 影響：
  影響 CLI 與 config 設計。

## 已知原則

- 不要憑空定義欄位含義

- 不要把未驗證結論寫成既定事實
