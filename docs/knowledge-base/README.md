# Knowledge Base

這個資料夾用來保存專案的交接資訊、決策脈絡、目前狀態與後續待辦。

## 文件順序

建議閱讀順序:
1. `next_agent_handoff.md`
2. `python_file_responsibilities.md`
3. `project_snapshot.md`
4. `decisions.md`
5. `open_questions.md`
6. `implementation_journal.md`

## 各文件用途

- `next_agent_handoff.md`
  下一位接手的人或 agent 應先看的交接摘要，包含目前狀態、重現指令、限制與下一步。
- `python_file_responsibilities.md`
  專案所有 Python 檔案的責任分工與快速定位表。
- `project_snapshot.md`
  專案目標、範圍與目前能力總覽。
- `decisions.md`
  已確認的技術決策與理由。
- `open_questions.md`
  尚未定案或需要再驗證的問題。
- `implementation_journal.md`
  實作過程中的時間序紀錄。

## 更新規則

當專案有重大調整時，至少同步更新:
1. `implementation_journal.md`
2. `next_agent_handoff.md`

如果有模組責任、檔案分工或入口更動，再同步更新:
1. `python_file_responsibilities.md`

如果有新的技術取捨或架構決策，再同步更新:
1. `decisions.md`
